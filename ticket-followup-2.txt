180 ₽/мес — приемлемо. Multi-bind и random allocation поняты.

Чтобы повысить вероятность получить IP из другого /16-префикса (поскольку выдача случайная и я не могу выбрать диапазон), решил параллельно проверить и другой регион. План такой:

1. **Закажу 2 новых IPv4 в зоне NSK-1** (через ваш UI) — кандидаты для смены IP на существующем сервере 5.129.207.254.

2. **Параллельно создам минимальный VPS в зоне SPB-3** (самый дешёвый тариф, мне нужен только TCP-открытый порт 22 для probe). Его IP уже будет привязан при создании — это даст 3-й кандидат **из другой зоны / другого пула IP-адресов**.

3. Все 3 кандидата (2 NSK + 1 SPB) прогоню через свою точку наблюдения в Yandex Cloud (probe каждые 5 минут) в течение 24-48 часов. По результатам выберу:
   - Если один из 2 NSK-IP стабилен → переключаю DNS на него, остальные отвязываю
   - Если оба NSK нестабильны, а SPB стабилен → миграция сервера в SPB-3
   - Если все 3 нестабильны → решаю отдельно (Cloudflare CDN или другой подход)

**Просьба к вам:** когда я закажу 2 NSK-IP, **привяжите их к серверу 5.129.207.254 батчем за один reboot** (вы предлагали такой вариант — "можем также привязать новый адрес за вас"). Если получится одним перезапуском вместо двух — идеально.

**Время reboot:** удобно ночью МСК / утром НСК, когда минимум клиентского трафика (~3-5 утра МСК = 7-9 утра НСК). Если у вас есть предпочтения по слоту — скажите.
